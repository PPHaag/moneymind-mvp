# MoneyMind UX Improvements Backlog

This document stores UX improvements that should be implemented later.

The goal is to capture ideas and feedback without interrupting current product development.

These improvements focus on clarity, tool positioning, and reducing perceived overlap between tools.

---

# Tool Positioning Improvements

Users sometimes perceive overlap between tools.

This is mainly a positioning and explanation issue, not a structural product issue.

Each tool should clearly answer one specific question.

### MoneyMind tool logic

1. Capital Map  
Where your capital sits today.

2. Allocation Check  
How your monthly cashflow is structured.

3. Wealth Leakage  
Where money quietly escapes.

4. Cost of Delay  
What waiting costs you.

5. Wealth Builder  
Where your wealth trajectory can go.

---

# Tool Explanation Layer

Each tool should include a short one-line explanation below the title.

Example:

Capital Map  
Understand where your capital sits.

Allocation Check  
See how your monthly cashflow is structured.

Wealth Leakage  
Identify money that quietly drains your wealth.

Cost of Delay  
See what postponing wealth building costs over time.

Wealth Builder  
Project your long-term wealth trajectory.

Example:

Understand → Diagnose → Correct → Accelerate → Build

Mapping to tools:

Understand → Capital Map  
Diagnose → Allocation Check  
Correct → Wealth Leakage  
Accelerate → Cost of Delay

This visual layer should help reduce perceived tool overlap.

# UX Consistency Goals

All tools should follow the same interaction structure:

Input → Calculate → Result → Insight → AI Insight

Consistency is more important than complexity.

The goal is to create a calm, intelligent, premium financial experience.
